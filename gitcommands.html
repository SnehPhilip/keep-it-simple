_____________________________
CONFIGURATION COMMANDS
_____________________________

Set user name
Code
git config --global user.name "Your Name"



Set user email
Code
git config --global user.email "your_email@example.com"


__________________________________
CREATING AND CLONING REPOSITORIES:
__________________________________

Initialize a new local repository.
Code
git init


[OR]

Clone an existing repository.
Code
git clone

___________________________
MAKING AND STAGING CHANGES:
[Navigate to your Git repository in the terminal: Code cd /path/to/your/repository]
___________________________

Check repository status.
Code
git status


Add a file to the staging area:
Code
git add file_name


Add all changes to the staging area:
Code
git add .


Commit staged changes with a message.
Code
git commit -m "Commit message"


Commit all tracked changes directly (skipping staging area):
Code
git commit -am "Commit message"


______________________________________________________________________________________________________________________
Why use git status ?
It gives
1. the Current Branch:
(The name of the branch you are currently on.)
2. the Changes to be committed:
(Files that have been staged using git add and are ready for the next commit.)
3. the Changes not staged for commit:
(Modified files in your working directory that have not yet been added to the staging area.
Untracked files:)
4. the New files in your working directory that Git is not currently tracking. These files will not be included in commits unless
you explicitly add them using git add.
5. the Information about ongoing operations:
If there's an ongoing rebase or merge, git status will provide details about its state.]
______________________________________________
WORKING WITH BRANCHES:
_________________________________________

list all local branches.
Code:
git branch


1. Create a new branch without switching to it:
Code:
git branch new_branch_name


2. Create a new branch and switch to it immediately:
Code:
git checkout -b new_branch_name


3. Switch to an existing branch.
Code:
git checkout branch_name


4. Merge a branch into the current branch:
Code:
git merge branch_to_merge


_____________________
DELETING LOCAL BRANCH
_______________________
Code :
git branch -d branch_name


______________________________________
SYNCHRONIZING WITH REMOTE REPOSITORIES:
_______________________________________
Fetch changes from a remote repository.
Code :
git fetch remote_name


Pull changes (fetch and merge) from a remote repository:
Code :
git pull remote_name branch_name


Push local commits to a remote repository:
Code :
git push remote_name branch_name


[NEVER PUSH CHANGES DIRECTLY TO REMOTE MASTER BRANCH, INSTEAD RAISE A PULL REQUEST IN GITHUB FOR REVIEW]
